# 工時追蹤系統 - 規格文件

## 1. 專案概述

### 1.1 目標
建立一個團隊工時追蹤網頁應用程式，讓團隊成員能夠記錄每週在各專案上投入的時間，並提供總覽與趨勢分析功能。

### 1.2 核心需求
- 使用者帳號系統（註冊、登入、登出）
- 工時記錄功能（記錄專案、時數、日期）
- 公開總覽頁面（所有人可見）
- 個人工時紀錄頁面
- 趨勢分析功能

---

## 2. 功能規格

### 2.1 使用者系統

| 功能 | 說明 |
|------|------|
| 註冊 | 使用者名稱、Email、密碼 |
| 登入 | Email + 密碼驗證 |
| 登出 | 清除登入狀態 |
| 個人資料 | 顯示/編輯使用者基本資訊 |

### 2.2 專案管理

| 功能 | 說明 |
|------|------|
| 新增專案 | 建立新的專案名稱 |
| 專案列表 | 顯示所有可用專案 |
| 專案狀態 | 啟用/停用專案 |

### 2.3 工時記錄

| 功能 | 說明 |
|------|------|
| 新增工時 | 選擇專案、輸入時數、選擇日期 |
| 編輯工時 | 修改已填寫的紀錄 |
| 刪除工時 | 移除錯誤的紀錄 |
| 週報檢視 | 以週為單位檢視工時 |

### 2.4 總覽頁面（公開）

| 功能 | 說明 |
|------|------|
| 團隊總覽 | 顯示所有成員在各專案的時數 |
| 週次篩選 | 選擇特定週次查看 |
| 專案統計 | 各專案總投入時數 |
| 成員統計 | 各成員總投入時數 |

### 2.5 趨勢分析

| 功能 | 說明 |
|------|------|
| 時間趨勢圖 | 顯示各專案時數隨時間的變化 |
| 專案比較 | 比較不同專案的時數分配 |
| 新專案標記 | 標示新加入的專案 |
| 個人趨勢 | 個人在各專案的時數變化 |

---

## 3. 資料模型

### 3.1 User（使用者）
```
{
  id: string
  username: string
  email: string
  password: string (hashed)
  createdAt: timestamp
}
```

### 3.2 Project（專案）
```
{
  id: string
  name: string
  description: string
  isActive: boolean
  createdAt: timestamp
}
```

### 3.3 TimeEntry（工時紀錄）
```
{
  id: string
  userId: string
  projectId: string
  hours: number
  date: date
  note: string (optional)
  createdAt: timestamp
  updatedAt: timestamp
}
```

## 5. 技術選型建議

### 5.1 前端
- **框架**: React + TypeScript
- **UI 元件**: Tailwind CSS
- **圖表**: Chart.js 或 Recharts
- **路由**: React Router
- **狀態管理**: React Context 或 Zustand

### 5.2 後端（未來實作）
- **選項 A**: Node.js + Express + PostgreSQL
- **選項 B**: Supabase（BaaS，快速開發）
- **選項 C**: Firebase（Google 生態系）

## 6. UI/UX 設計原則

### 6.1 設計風格
- 簡潔現代的介面
- 以卡片式設計呈現資訊
- 響應式設計（支援手機/平板/桌面）

### 6.2 色彩方案
- 主色: 藍色系 (#3B82F6)
- 成功: 綠色 (#10B981)
- 警告: 黃色 (#F59E0B)
- 錯誤: 紅色 (#EF4444)
- 背景: 淺灰 (#F3F4F6)

### 6.3 使用者體驗
- 快速填寫工時（最少點擊次數）
- 清楚的視覺回饋
- 直覺的導航結構
